# 0001 — Kubernetes как оркестратор Linux-компонентов

Дата: 2026-07-16
Статус: принято

## Контекст

Нужен оркестратор для API server, RabbitMQ, PostgreSQL, вспомогательных
сервисов-заглушек.

## Решение

Используем Kubernetes для всех Linux-компонентов. Готовые операторы
(RabbitMQ Cluster Operator, CloudNativePG/Patroni) закрывают большую часть
эксплуатационной сложности (кластеризация, failover, backup).

## Последствия

- Требуется поддерживать Helm-чарты на каждый сервис.
- Windows NX-воркер не подчиняется этому решению — см. ADR 0002.
